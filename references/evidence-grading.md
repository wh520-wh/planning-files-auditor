# Evidence Grading

> Extracted from SKILL.md '## 证据规则' section for reference (loaded on demand).


使用严格的证据分级：

| 证据强度 | 示例 | 处理方式 |
|---|---|---|
| 强 | 源码、测试、通过/失败命令输出、git diff、日志、配置、与行为一致的文档 | 可以验证或反驳声明 |
| 中 | 已有构建日志、运行日志、截图、生成产物、当前会话终端输出 | 一致时可以支持声明 |
| 弱 | `progress.md` 记录、未复现的命令声明、状态标签、注释里的“已完成” | 只能当作声明，不能当作证明 |
| 无 | 没有对应代码、没有输出、没有产物、没有测试 | 标记为 `未验证` |

不要因为一条弱声明听起来很具体，就把它升级为已验证。
